## 1. 产品概述

### 1.1 产品名
**Leslie-stock** — A 股 + 港股 AI 价值投资助手

### 1.2 一句话定位
**让每个散户都有一个"段永平 + 巴菲特"在身边做投研决策**。

### 1.3 核心价值主张

| 价投散户的痛点 | Leslie-stock 的答案 |
|---|---|
| 不知道该看哪些股票 | **推荐榜**：按 BG 框架对能力圈内股票综合打分排序 |
| 看不懂财报、不会建立框架 | **6 维度评估**：商业模式 / 护城河 / 管理层 / 财务质量 / 估值 / 能力圈 |
| 每天被市场噪音干扰 | **GLM-5.1 解读**：自动过滤噪音、识别真信号、标记卖出条件 |
| 没人提醒持仓基本面变化 | **每日盘后简报** + 邮件 + 网站推送 |
| 不知道何时该卖 | **卖出三条件**（段永平）：当初买错 / 基本面变坏 / 找到明显更好的 |

### 1.4 设计哲学（不可妥协）

1. **方法论内核而非工具堆**：所有功能都要回归 BG_DNA 框架（800 行内化为系统 prompt）
2. **量化做硬数据，AI 做软判断**：13 项财务红线机器算，3 个定性维度 GLM 填
3. **去神化、不预测**：不预测大盘、不喊牛市熊市、不打榜热点
4. **能力圈优先**：圈外的股票主动过滤，不替用户做圈外决策
5. **缓存优先**：分析过的存下来，按需重算

## 2. 用户画像

### 2.1 主用户：价投散户（90% 流量）

- **画像**：25-50 岁，资产 10 万-500 万，中长线（年级别）持仓
- **行为**：周末看雪球 / 看年报，工作日盘后看持仓
- **认知水平**：知道段永平 / 巴菲特，能看懂 ROE / PE / 自由现金流，但建不起完整框架
- **典型持仓**：茅台、宁德、腾讯、网易、平安、招行、中海油等核心资产

### 2.2 次要用户：财经从业者（10%）

- 投顾、私募基金研究员、金融专业学生
- 用 Leslie-stock 做候选池初筛 + 撰写研报辅助
- 未来 B 端付费可能性

### 2.3 明确不服务的用户

- 短线 / 日内交易者
- 题材 / 板块炒作派
- 量化纯算法用户（他们有 Wind / 同花顺 iFinD）

## 3. 现状与差异化

### 3.1 竞品扫描

| 工具 | 优势 | 缺陷 | Leslie-stock 比它强在哪 |
|---|---|---|---|
| **同花顺问财** | 自然语言筛选、流量大 | 只做量化、无定性框架 | BG 6 维度 + GLM 定性 |
| **雪球** | 社区氛围、价投圈活跃 | 信息散、无系统输出 | 系统化报告 + 每日简报 |
| **东方财富** | 数据全、用户多 | 用户要自己消化 | 一键 AI 报告 |
| **Wind / iFinD** | 数据 + 工具完善 | B 端定价 + 散户不友好 | 散户友好定价 + 价投视角 |
| **ChatGPT / Claude** | LLM 能力强 | 不懂中国股票实时数据 | 数据 + LLM + 框架内化 |

### 3.2 我们的护城河（按强度排序）

1. 🟢 **BG_DNA 方法论内化**（最强）：800 行段永平 + 巴菲特方法论作为系统 prompt，每一次决策都按它走
2. 🟢 **量化 + AI 混合架构**：硬指标机器算（快、准），软判断 LLM 填（深、有人味）
3. 🟡 **数据接入工程**：curl_cffi 反爬 + 多源 fallback（不需要付费数据也能做 MVP）
4. 🟡 **专注 A 股 + 港股**：中国散户主战场，海外工具无法竞争
5. 🔴 **品牌 + 内容**：起步阶段，靠社区运营建立

## 4. 功能模块（v0.1 全部已交付）

### 4.1 推荐榜（首页 `/`）

**用户故事**：进入应用第一眼，看到的是"今天该看哪些股票"，不是"我的持仓涨跌"。

**功能清单**：
- 综合 TOP 20（按 BG 综合得分排序）
- 4 个能力圈分榜：AI 上下游 / 能源-材料-资源 / 医药生物 / 互联网
- 估值便宜榜（PE < 15）
- 财务过硬榜（财务 ≥ 80 分）
- **AI 徽章** ✨ GLM-5.1：已用 LLM 深度分析的股票

**数据源**：data/rankings.json（rank_universe.py 每日自动生成）

### 4.2 全局搜索（顶部）

**用户故事**：能力圈很大，推荐榜不够，需要直接搜任意股票。

**功能**：
- 模糊匹配：代码 / 中文名 / 行业
- 键盘上下选 + 回车跳转
- 按市值排序结果

**数据源**：data/universe.json（8100+ 只股票）

### 4.3 个股深度页 `/stock/[code]`

**用户故事**：选好一只股票后，看完整 BG 报告。

**功能**：
- BG 6 维度评分（商业模式 / 护城河 / 管理层 / 财务质量 / 估值 / 能力圈）
- **段永平 / 巴菲特视角 verdict**（GLM-5.1 输出，2-3 句话观点）
- 5 年财务走势图表（ROE / 毛利率 / 净利率 / 资产负债率，含红线参考）
- 实时行情 + 估值
- **缓存机制**：分析过的存 JSON，24 小时内直接读；点"🔄 重新生成"强制刷新
- **按需触发**：未分析过的股票点"🤖 生成 BG 深度分析"按钮，GLM 后台跑 60-90 秒

### 4.4 持仓监控 `/portfolio`

**用户故事**：买入后我希望系统主动提醒持仓基本面变化和卖出信号。

**功能**：
- 持仓清单（来自 core/portfolio.csv，手动维护）
- **每只持仓的 AI 卡片**：
  - BG 综合得分
  - 段永平 / 巴菲特视角 verdict
  - 最近 7 天新闻的 GLM 解读（整体信号 🟢🟡🔴）
  - 建议方向（持有 / 加仓 / 减仓 / 卖出 / 观望）
  - 卖出条件触发检查
- 你自定义的"卖出条件清单"
- 盘后简报（markdown 渲染）

### 4.5 BG 评估引擎（核心算法）

详见 `core/BG_DNA.md`。简版：

**6 维度**：
1. 商业模式（GLM）：10 年后还在赚钱吗？长坡厚雪？
2. 护城河（GLM）：5 种类型，过去 5 年宽了还是窄了？
3. 管理层（GLM）：诚信 + 理性 + 股东导向
4. 财务质量（机器）：13 项红线 — ROE/FCF/负债率/商誉等
5. 估值（机器）：PE 历史分位 / 十年回本期 / 安全边际
6. 能力圈（机器）：行业关键词匹配 Leslie 4 个起点圈

**评分逻辑**：
- 每维度 0-100，按权重加权
- **一票否决**：任一维度 < 60 → 综合"过滤"
- 综合 ≥ 80 → 重点候选；65-79 → 学习池；<65 → 过滤

### 4.6 新闻聚合 + GLM 解读

**功能**：
- 东财搜索 API 直拉（绕过 akshare regex bug）
- 巨潮 cninfo 公告（A 股权威源）
- GLM-5.1 按 BG 框架解读每条：
  - 标签：商业模式 / 护城河 / 管理层 / 财务 / 行业政策 / 估值 / **市场噪音**
  - 信号强度：🟢加分 / 🟡警告 / 🔴红色 / ⚪噪音
- 整体总结：narrative + action_suggestion + sell_condition_triggered

**真实例子**（茅台过去 7 天）：
> "过去 7 天茅台的核心护城河未受损伤，商标续签维持 1.5% 费率虽是历史遗留瑕疵但未恶化，股价下跌与资金流出纯属市场情绪。"

### 4.7 自动化（macOS launchd）

每日 16:30 自动跑：
1. 更新 universe（8100 只股票）
2. BG 排名 + GLM-5.1 TOP 30 深度分析
3. 拉持仓股新闻 + GLM 解读
4. 生成 + 发送每日简报（Gmail SMTP）

## 10. 风险与限制

### 10.1 数据合规

- ✅ 不直接转售付费数据（Wind / iFinD）
- ✅ 不爬取付费墙内容
- ⚠️ akshare / 东财属于灰色区域，长期商用必须迁移到付费 API
- ✅ 用户数据加密（持仓属于隐私）

### 10.2 投资建议法律风险

- ✅ 所有 GLM verdict 标记"参考性质，非投资建议"
- ✅ 不提供具体仓位 / 价格指令
- ✅ 用户每次买入要自己写"买入备忘录"（BG_DNA 规定）
- ⚠️ 若做 B 端投顾业务，需要持牌（证券投资咨询资格）

### 10.3 LLM 幻觉

- ⚠️ GLM-5.1 偶尔会编造数据
- 缓解：**所有数字必须来自机器算的财务部分**，LLM 只做定性
- 缓解：用户可一键标记错误，反馈进入提示词优化
- 缓解：定期抽检（每月 10-20 只股票人工 review）

### 10.4 数据源稳定性

- ⚠️ 东财反爬规则动态变化（curl_cffi 偶尔失效）
- 缓解：多 fingerprint 轮换 + 定期更新
- 长期：迁移到付费 API（Tushare Pro / 同花顺）

### 10.5 监管风险

- ⚠️ A 股投资建议在中国受证监会监管
- 缓解：明确"工具属性"，不做"推荐买入"措辞
- 缓解：B 端业务通过持牌机构合作
