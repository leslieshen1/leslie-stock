# 盘报 Playbook —— 固定提示词 + 模板(盘前 / 盘中 / 收盘)

> 这份是唯一标准。每次写盘报都照这个来,不再临场发挥。
> 写完 → 用户 OK → `publish_report.py` 推到 `/reports` tab。

---

## 一、固定提示词(口吻 / 数据 / 合规)—— 每篇都遵守

**口吻(Leslie)**
- 锐利、有框架、反共识、说人话;把美股当"正和合约游戏"在划重点。
- 不喊单、不给目标价、不保证。是「该看什么」,不是「买什么」。
- 中文。锋利但不油腻;每个判断后面要有"为什么"。

**出报时点(铁则 —— 2026-06-11 定,Leslie 问"是不是该完全收盘才出"后立的)**
- 美股一天三段(ET):盘前 04:00–09:30 → 正股 09:30–16:00 → **盘后 16:00–20:00**。
- **收盘报告 + 收盘卡:默认北京 08:00(美东 20:00)之后出** —— 盘后定格,ORCL 那种盘后雷才有终值。
- 北京 04:00–08:00 之间被要求出报:可以出,但页脚必须标 **「盘后仍在交易中 · 截至 HH:MM ET」**,盘后数字按快照写。
- 盘前报告不变:开盘(北京 21:30)前 ~1h,即 20:30 北京。

**数据(只认 Nasdaq 实时,逐只 `/api/quote/{sym}/info`)**
- **盘前**:对比基准 = **上一交易日收盘**(secondaryData);pm 涨跌 = primaryData。
- **盘中 / 收盘**:涨跌 = 当日(primaryData percentageChange)。
- **盘后定格数(20:00 ET 后)**:`/api/quote/{sym}/extended-trading?assetclass=stocks&markettype=post` → `infoTable.rows[0].consolidated`(此时 info 端点的 primary 已滚回日内收盘,拿不到盘后)。
- 指数用 ETF 代理:SPY=标普 · QQQ=纳指 · DIA=道指 · IWM=罗素。
- **必查两个日历(别再漏宏观 —— 2026-06-09 漏了 CPI 的教训):**
  - 宏观:Finnhub `calendar/economic` → CPI / PPI / PCE / FOMC / 非农 / 失业金,**带日期+时间(ET)**。
  - 财报:Finnhub `calendar/earnings` → 今日盘前/盘后 + 明日重头,**公司名没有就只写代码、绝不猜**。

**标的范围(固定)**
- **只两类:大标的 + 热门标的。** 不堆全市场。
- 大标的 = 指数权重 mega(AAPL/MSFT/NVDA/GOOGL/AMZN/META/AVGO/TSLA + JPM/V/LLY/WMT)。
- 热门标的 = AI 半导体 / 光通信(MU/MRVL/ACMR/AXTI/AAOI/ARM/SMCI/AMD/TSM/NVTS)+ 加密 proxy(MSTR/COIN)+ 太空(ASTS/LUNR)+ 当日 top movers。**MU、NVDA 这类龙头每篇必含。**

**合规(固定页脚)**
> _<快照时间> · 大标的 + 热门标的 · 我不是股神 · 非投资建议_
- 教育 / 框架,非荐股;不面向中国大陆。

## 二、三种模板(骨架固定,顺序不变)

公共骨架:**定调 → 指数 → 大标的 → 热门标的 → [类型专属] → 收尾**。

## 三、发布流程("OK 就推")

1. 我照模板写好 → 贴给你。
2. 你说 **OK**。
3. 我把 markdown 存到 `data/reports/_draft.md`,跑:
   ```
   uv run python scripts/publish_report.py --type {premarket|intraday|close} \
     --time "HH:MM ET" --title "..." --file data/reports/_draft.md
   ```
   → 追加进 `web/public/data/reports.json`(最新在前)。
4. `cd web && vercel --prod --yes --archive=tgz` → 出现在 **/reports tab**。

类型标签:`premarket`=盘前 · `intraday`=盘中 · `close`=收盘。
